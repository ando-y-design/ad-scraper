# 工程④　データ保存（SQLite / Google Sheets）

## 概要

`writer_worker` が `result_queue` を消費し、**SQLite** にINSERTしつつ **Google Sheets** に追記する。

---

## 保存フロー

```
result_queue から会社情報 dict を取得
         │
         ▼
 ┌── 重複チェック（SQLite） ─────────────────────────────┐
 │  base_url（ドメイン）が companies テーブルに存在？       │
 │  ├─ 存在する → スキップ                               │
 │  └─ 存在しない → INSERT へ                            │
 └──────────────────────────────────────────────────────┘
         │
         ▼
 ┌── SQLite INSERT ──────────────────────────────────────┐
 │  companies テーブルに1行追加                           │
 │  WALモードで並列書き込みに対応                         │
 └──────────────────────────────────────────────────────┘
         │
         ▼
 ┌── Google Sheets 追記 ─────────────────────────────────┐
 │  sheets_writer.append_row() で最終行に1行追記          │
 │  Service Account 認証（credentials.json）             │
 └──────────────────────────────────────────────────────┘
```

## Google Sheets 連携

### 認証設定

1. GCP でサービスアカウントを作成
2. Google Sheets API を有効化
3. 秘密鍵JSONを `credentials.json` として配置
4. 対象スプレッドシートにサービスアカウントのメールを共有

### config.json の設定

```json
"google_sheets": {
  "sheet_id": "1hf4q3YC3rYSB5jrceQ3...",
  "service_account_key_path": "credentials.json"
}
```

### カラム構成（現行）

| A列 | B列 | C列 | D列 | E列 | F列 |
|-----|-----|-----|-----|-----|-----|
| 会社名 | LP URL | 電話番号 | キーワード | 広告ソース | 取得日 |

### ヘッダー自動修復

`writer_worker` は起動時にSheets1行目のヘッダーを確認し、カラム定義と不一致なら自動で書き直す。

---

## DBゴミクリーナー（db_janitor）

watchdog が**10分ごと**に `utils/db_janitor.py` を呼び出し、以下のゴミデータを自動削除する：

- 会社名・電話番号ともに空のレコード
- 明らかに不正なURL（`javascript:`, `data:` 等）
- 重複ドメインの古いレコード

## データ確認コマンド

```python
import sqlite3
conn = sqlite3.connect('companies.db')

# 総件数
conn.execute('SELECT COUNT(*) FROM companies').fetchone()

# 今日の取得件数
conn.execute("SELECT COUNT(*) FROM companies WHERE found_date = date('now','localtime')").fetchone()

# 最新10件
conn.execute('''
    SELECT company_name, phone, ad_sources, found_date
    FROM companies ORDER BY id DESC LIMIT 10
''').fetchall()
```

## よくある問題と対処

| 症状 | 原因 | 対処 |
|------|------|------|
| Sheets書き込みエラー | 認証切れ / API無効 | `credentials.json` を再発行 |
| 重複データが登録される | base_url の正規化ミス | `normalizer.py` の URL正規化ロジックを確認 |
| DBサイズ肥大 | ゴミデータの蓄積 | `utils/db_janitor.py` を手動実行 |
| Sheets API 429 | 書き込みレート超過 | `writer_worker.py` のretry待機時間を調整 |
