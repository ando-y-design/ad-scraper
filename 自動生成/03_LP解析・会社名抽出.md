# 工程③　LP解析・会社名抽出

## 概要

`processor_worker` が8並列で `lp_queue` を消費し、各LPから **会社名・電話番号** を抽出して `result_queue` に投入する。

---

## 解析フロー

```
lp_queue から LP URL を取得
         │
         ▼
 ┌── LP 取得 ──────────────────────────────────────────────┐
 │  requests + BeautifulSoup でHTMLを取得                   │
 │  ブロックドメインはスキップ（config.json filters 参照）  │
 └──────────────────────────────────────────────────────────┘
         │
         ▼
 ┌── 特商法ページ解析 ──────────────────────────────────────┐
 │  company_finder.find_company_info(url, html)             │
 │  1. 特商法ページへの内部リンクを探す                     │
 │     (/tokutei, /legal, /tokuteishohо, /company 等)      │
 │  2. 特商法ページのHTMLから会社名テキストを抽出           │
 │  3. 見つからなければ LP本体のテキストから推定             │
 └──────────────────────────────────────────────────────────┘
         │
         ▼
 ┌── 法人名正規化 ──────────────────────────────────────────┐
 │  normalizer.normalize_company_name()                     │
 │  - 「株式会社〇〇」「〇〇株式会社」の揺れを統一          │
 │  - 記号・スペース除去                                    │
 └──────────────────────────────────────────────────────────┘
         │
         ▼
 ┌── 英語法人名 → 国税庁API変換（オプション）──────────────┐
 │  legal_name_resolver.resolve_legal_name()                │
 │  "M-Style Japan Co., Ltd." → "M-Style Japan株式会社"    │
 │  ※ nta_api_key が有効な場合のみ                         │
 └──────────────────────────────────────────────────────────┘
         │
         ▼
 ┌── 電話番号抽出 ──────────────────────────────────────────┐
 │  phone_finder.find_phone_numbers(html)                   │
 │  - 正規表現で日本の電話番号を全件抽出                    │
 │  - strategy=sme: 中小企業向け（0120/0800以外を優先）     │
 │  - 複数ある場合は代表番号を選定                          │
 └──────────────────────────────────────────────────────────┘
         │
         ▼
 result_queue に投入 → writer_worker へ
```

## 会社名抽出の仕組み

### company_finder.py の探索順

1. **特商法ページ直接探索**
   - LP のHTML内からリンクを走査し、特商法ページURLを特定
   - 特商法ページを取得し「販売業者」「運営会社」「会社名」等のラベルに続くテキストを取得

2. **フォールバック: フッター・会社情報エリア**
   - `<footer>`, `class="company"`, `id="about"` 等の要素から会社名パターンを検索

3. **最終フォールバック: メタタグ**
   - `<meta name="author">`, OGP タグから推定

### normalizer.py の正規化ルール

| 処理 | 例 |
|------|-----|
| 前後スペース除去 | `" 株式会社ABC "` → `"株式会社ABC"` |
| 全角スペース除去 | `"株式会社　ABC"` → `"株式会社ABC"` |
| 法人格の位置統一 | `"ABC株式会社"` はそのまま保持 |
| 記号の除去 | `"株式会社ABC・商会"` → `"株式会社ABC商会"` |

## 電話番号抽出の仕組み

### phone_finder.py のマッチングパターン

```
固定電話: 03-XXXX-XXXX / 0X-XXXX-XXXX
携帯:     090-XXXX-XXXX / 080-XXXX-XXXX / 070-XXXX-XXXX
フリーダイヤル: 0120-XXX-XXX / 0800-XXX-XXXX
IP電話:   050-XXXX-XXXX
```

### strategy=sme（中小企業向け）の優先順

1. 市外局番付き固定電話（最優先 → テレアポに使えるため）
2. 携帯電話番号
3. フリーダイヤル・IP電話（最後）

---

## 国税庁API連携（nta_lookup.py）

| 状態 | 動作 |
|------|------|
| APIキー有効 | 会社名を国税庁DBと照合して正式法人名に補正 |
| APIキー404 | セッション中の全API呼び出しを停止（ログ1回のみ） |
| APIキー未設定 | スキップ（デバッグログのみ） |

**APIキーの取得**: https://www.houjin-bangou.nta.go.jp/webapi/

---

## 重複排除

`writer_worker` 側で `base_url`（ドメイン部分）をキーにSQLiteで重複チェック。同一ドメインの企業は再登録しない。

## 主要ファイル

| ファイル | 役割 |
|---------|------|
| `workers/processor_worker.py` | 並列解析ワーカー（8スレッド） |
| `processors/company_finder.py` | 特商法ページ解析・会社名抽出 |
| `processors/legal_name_resolver.py` | 英語法人名→国税庁API変換 |
| `processors/normalizer.py` | 会社名・電話番号正規化 |
| `processors/phone_finder.py` | 電話番号抽出 |
| `utils/nta_lookup.py` | 国税庁法人番号API クライアント |
| `utils/nta_local_db.py` | ローカル法人DBフォールバック |

## よくある問題と対処

| 症状 | 原因 | 確認場所 |
|------|------|---------|
| 会社名が空欄 | 特商法ページへのリンクなし | `processors/company_finder.py` の find_company_info() |
| 電話番号が空欄 | 電話番号のHTML埋め込みなし（画像等） | `processors/phone_finder.py` の正規表現を確認 |
| NTA API 警告連発 | APIキーが未有効化 | `config.json` の `nta_api_key` を再発行 |
| 抽出精度が低い | ブロックドメインのすり抜け | `config.json` の `filters.blocked_domains` に追加 |
